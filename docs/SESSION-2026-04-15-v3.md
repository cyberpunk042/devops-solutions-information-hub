### Source-mechanism refactor — completed

**Directive logged:** `raw/notes/2026-04-15-directive-project-based-source-mechanism-with-git-fallback.md`

**Schema extension (`wiki/config/wiki-schema.yaml`):** Added `project` to `source_needs_one_of` alongside `url` and `file`. Added `source_project_companion_field: path` declaration — when source has `project`, `path` is required.

**Registry extension (`wiki/config/sister-projects.yaml`):** All 5 sister projects now carry a `remote:` block with `type: github, owner: cyberpunk042, repo: <name>, default_branch: main, url: <GitHub URL>`. All local `path:` fields switched from absolute `/home/jfortin/...` to portable `~/` home-expansion form. AICP gained `aliases: [devops-expert-local-ai]` so sources can cite either name. Added `source_resolution` block declaring default backend order `[local, gh_api, raw_github]` + 1h cache TTL.

**Resolver module (`tools/source_resolver.py`, NEW):** 3-backend fallback chain returning structured Receipts (Adapters-Never-Raise pattern). Backend 1 reads from local filesystem. Backend 2 invokes `gh api repos/{owner}/{repo}/contents/{path}` (checks `gh auth status` first). Backend 3 falls back to `https://raw.githubusercontent.com/{owner}/{repo}/{branch}/{path}` via `urllib`. Each failure returns structured per-backend receipt; the aggregate receipt carries `tried[]` list. Includes legacy path inference for gradual migration. CLI: `python3 -m tools.source_resolver <project> <path>`.

**Validation extension (`tools/validate.py`):** Two new checks: (1) `source_project_missing_path` — error if `project:` present without `path:`; (2) `source_file_absolute_path_deprecated` — warning on legacy `/home/<user>/...` paths, directing migration to project+path form.

**Migration:** Auto-migrated all 29 absolute-path source entries across 10 wiki pages from `file: /home/jfortin/<project>/<rest>` to `project: <name>, path: <rest>` via one-shot Python script. Result: 0 remaining absolute-path sources, 0 deprecation warnings.

**Consumption tracker updated (`tools/sister_project.py`):** `_build_consumption_index` now parses structured frontmatter `sources:` for `project + path` pairs in addition to existing absolute/relative regex matching. Registry-key lookup added to handle projects whose directory basename differs from registry key (AICP case — basename `devops-expert-local-ai`, registry key `aicp`). Also respects declared `aliases` in the registry.

**Verification:** all 4 sisters still absorb correctly (openarms=14, openfleet=3, aicp=1, control-plane=3). Pipeline PASS. Resolver smoke tested against 2 local + 1 missing (which correctly tried all 3 backends and returned aggregate failure receipt).

**Remote-backend smoke test:** forced `--backends gh_api raw_github` (skip local) on a known-live openarms file — `gh_api` succeeded (13.3 kB retrieved via authenticated `gh api`). Forced `--backends raw_github` alone returned HTTP 404. **Empirical finding: the 5 sister repos are PRIVATE on GitHub**, so `raw_github` is effectively unusable without auth tokens in URLs. The gh_api backend is the practical remote fallback for this ecosystem. Documented in `sister-projects.yaml` comments so future readers know the policy shape. raw_github kept in the chain anyway for public-repo content (external sources, open-source references outside our ecosystem).
