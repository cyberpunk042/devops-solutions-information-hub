**Resolver module (`tools/source_resolver.py`, NEW):** 3-backend fallback chain returning structured Receipts (Adapters-Never-Raise pattern). Backend 1 reads from local filesystem. Backend 2 invokes `gh api repos/{owner}/{repo}/contents/{path}` (checks `gh auth status` first). Each failure returns structured per-backend receipt; the aggregate receipt carries `tried[]` list. Includes legacy path inference for gradual migration. CLI: `python3 -m tools.source_resolver <project> <path>`.

**Validation extension (`tools/validate.py`):** Two new checks: (1) `source_project_missing_path` — error if `project:` present without `path:`; (2) `source_file_absolute_path_deprecated` — warning on legacy `/home/<user>/...` paths, directing migration to project+path form.

**Consumption tracker updated (`tools/sister_project.py`):** `_build_consumption_index` now parses structured frontmatter `sources:` for `project + path` pairs in addition to existing absolute/relative regex matching. Registry-key lookup added to handle projects whose directory basename differs from registry key (AICP case — basename `devops-expert-local-ai`, registry key `aicp`).
