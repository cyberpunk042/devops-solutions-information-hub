# Model Guide: Second Brain

## Summary

The Second Brain model describes how to build a personal knowledge management system that compounds over time rather than decaying into an abandoned folder. It synthesizes two dominant methodologies — PARA (optimized for action and resource management) and Zettelkasten (optimized for idea development and connection discovery) — into a hybrid architecture extended by LLM-assisted maintenance. The model covers the structural principles (atomic notes, heterarchical linking, progressive distillation), the tools that implement them (Obsidian as the vault, Obsidian CLI and skills for programmatic interaction), and why LLM automation is the specific innovation that eliminates the maintenance problem that has killed every previous personal wiki attempt.

## Prerequisites

- A desire to retain and build on knowledge across time, not just within a single session
- Willingness to invest in a structured capture and organization habit
- No prior PKM experience required — the model addresses common failure modes from first principles

## Outcomes

After completing this learning path you will understand:

- Why PARA and Zettelkasten address different halves of the same problem, and why the hybrid model works better than either alone
- The three note lifecycle stages (fleeting → literature → permanent) and how to apply them to any source material
- Why the maintenance problem is the specific failure mode that has killed every previous wiki attempt, and how LLM automation eliminates it
- How Obsidian functions as knowledge infrastructure: graph view, backlinks, CLI programmatic access, and skills integration
- How the progressive distillation pattern maps to this wiki's maturity ladder (seed → growing → mature → canonical)
- How this wiki itself is a hybrid PARA+Zettelkasten system extended with LLM maintenance at near-zero cost
