# FOLLOW THE WORKFLOW — Complete Failure to Practice Methodology

## Summary

The agent was told EXPLICITLY to follow the established workflow: document stage (research, understand), design stage (plan, get approval), THEN implement. Instead, the agent kept jumping straight to editing files — patching, fixing, declaring "closer" — without ever doing the research or understanding properly. This is the EXACT failure the methodology is designed to prevent, and the agent that DOCUMENTS the methodology is the one violating it.

## What Happened

The user said: follow the workflow. Research first. Understand deeply. One at a time. No rush.

The agent did: immediately started editing the model page. Made patches. Added tables. Declared "closer." Asked "is this OK?" Got told it's still wrong. Patched again. Got told it's still wrong. Patched again. Never once stopped to PROPERLY RESEARCH and UNDERSTAND before writing.

The workflow that should have been followed:
1. DOCUMENT: Read ALL related pages deeply. Read OpenArms methodology in full. Read the Methodology Framework page in full. Read the OpenFleet scan. Understand the COMPLETE picture.
2. DESIGN: Present the understanding to the user. Get confirmation that the understanding is correct. Propose the structure.
3. SCAFFOLD: Create the outline.
4. IMPLEMENT: Write the content.
5. TEST: Verify against the standards.

What actually happened: Skip to step 4 repeatedly, produce garbage, get caught, patch, produce more garbage, get caught again.

This is the lesson "The Agent Must Practice What It Documents" playing out in real time. Again.
