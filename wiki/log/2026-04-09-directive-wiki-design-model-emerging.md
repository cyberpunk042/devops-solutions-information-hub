# Wiki Design Model Emerging — Obsidian Styling as Standard

## Summary

The Obsidian callout styling applied to the Methodology model page establishes a new kind of standard — not content structure (that's the schema) or content quality (that's the standards page) but visual DESIGN. This is a start, not finished, but it's heading toward a clear model of wiki design that includes how pages LOOK and FEEL, not just what they contain.

## Operator Directive (verbatim)

> see we now have established another kind of high standard but one more generic for the obsidiant / advanced markdown style. not a finished standard but a start that will lead to a clear model of wiki design

## Interpretation

Three layers of standards are emerging:
1. **Content structure** — the schema (wiki-schema.yaml) defines types, fields, required sections
2. **Content quality** — the standards page defines what "good" looks like per type
3. **Visual design** — the callout vocabulary defines how pages LOOK in Obsidian

This third layer is NEW and needs its own model/standard document. It will define:
- Which callout types to use for which purpose
- Consistent visual patterns across all page types
- How tables, code blocks, and diagrams should be formatted
- CSS snippets for custom callout types if needed
- How the styling degrades gracefully in non-Obsidian renderers

This is a start that will grow into a wiki design model — potentially becoming the 15th named model.
