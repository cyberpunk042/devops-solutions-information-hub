# E017 — Context Engineering Framework — Formalized as Model with Standards
## Summary

Formalize context engineering as its own MODEL in the wiki — with standards, examples, and practices. Currently scattered across lessons (Structured Context, Five Cognitive Contexts) and patterns (Validation Matrix, Tier-Based Depth). This epic consolidates into: Model: Context Engineering (how to design context injections), Context Engineering Standards (what good injections look like), and practical examples (CLAUDE.md structure, skill templates, task context templates, heartbeat templates). Covers: prompt engineering, context engineering, structured context as proto-programming, tiers and context capacity, smart autocomplete chains.

## Operator Directive

> "also things like the structure and format and context engineering and prompt engineering and smart autocomplete chain, and tiers and size of context capacity"

> "Markdown offer proto-programming and proto-programming is what AI does best"

> "If throughout our injections we use the proper formats adapted to the proper situation, not only we can connect with the information programmatically but there is a natural structure and consistency and pattern that can emerge in the AI context"
