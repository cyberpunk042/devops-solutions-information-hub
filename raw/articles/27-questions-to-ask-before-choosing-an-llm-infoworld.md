## All 27 Questions

### Performance & Technical Specs

1. **Model Size** — Parameter count indicating encoded information capacity
2. **Hardware Compatibility** — Whether the model fits and runs on available infrastructure
3. **Time to First Token (TTFT)** — Speed for interactive, real-time applications
4. **Rate Limits** — Processing capacity constraints from providers or hardware
5. **Context Window Size** — Maximum prompt length the model can handle

### Capabilities & Features

6. **Reasoning vs. Speed Balance** — Tradeoff between iterative thinking and response time
7. **Model Stability** — Consistency in avoiding divergent, nonsensical outputs
8. **Knowledge Cutoff Date** — Training data recency for embedded facts
9. **Fine-tuning Support** — Ability to conduct domain-specific additional training
10. **Supported Media Types** — Input/output formats (text, images, PDFs, etc.)
11. **Prompting Structure** — How the model responds to different prompt formats
12. **Tool Use Capability** — Integration with external functions, databases, APIs
13. **Agentic Features** — Multi-model coordination and reasoning capabilities

### Licensing & Availability

14. **Open Source Status** — Access to model weights and freedom to modify
15. **Guaranteed Lifespan** — Commitment to ongoing support and availability
16. **Batch Architecture Support** — Discounted delayed processing options

### Cost & Environment

17. **Pricing Model** — Cost structure relative to usage frequency
18. **Environmental Impact** — Electricity and water consumption per response
19. **Renewable Energy Use** — Whether infrastructure uses clean power sources

### Legal & Compliance

20. **Training Data Provenance** — Whether synthetic data was used in development
21. **Copyright Issues in Training Set** — Potential pirated or unlicensed content
22. **Third-party Audit Availability** — Independent verification of training data
23. **Indemnification Guarantees** — Contractual protection against infringement claims
24. **Compliance Requirements** — SOC2, HIPAA, GDPR adherence capabilities
25. **Geographic Location** — Physical server location relative to data regulations

### Implementation Considerations

26. **Human-in-the-Loop Support** — Capability for human oversight in reasoning processes
27. **Model Quirks** — Distinctive behavioral patterns affecting usability

## Core Thesis

"Not every application requires the same support" — comprehensive evaluation is essential for matching models to specific use cases. The 27 questions span 6 categories (performance, capabilities, licensing, cost/environment, legal, implementation).
