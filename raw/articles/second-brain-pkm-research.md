# Second Brain / PKM Research Notes

Compiled: 2026-04-08
Sources: zettelkasten.de, buildin.ai, Obsidian forum, web search

## Core PKM Methodologies

### 1. PARA (Building a Second Brain — Tiago Forte)
- Projects, Areas, Resources, Archives
- Action-oriented filing, not topic-based
- CODE workflow: Capture → Organize → Distill → Express
- Progressive summarization: highlight → bold → executive summary
- Strength: resource management, actionability, project focus
- Weakness: never extracts ideas from source materials

### 2. Zettelkasten (Niklas Luhmann)
- Atomic notes: one idea per note, self-contained
- Heterarchical network: connections between notes, not folders
- "Language of knowledge" vs PARA's "language of action"
- Strength: deep knowledge processing, reveals connections, supports long-term research
- Weakness: can feel abstract, harder to manage tasks/projects

### 3. Recommended Hybrid (Zettelkasten + BASB)
- PARA for organizing resources and task management
- Zettelkasten for processing materials into atomic notes
- "BASB manages exactly what is outside the Zettelkasten"
- Iceberg concept: PARA's lower layers (areas, resources) remain stable beneath projects
- At project start: structure note in ZK → collect → search → link
- At project completion: review → feedback to ZK → archive in PARA

## Relevance to This Wiki

This wiki already implements many of these principles:
- Atomic pages (one concept per page) = Zettelkasten atomic notes
- Relationships section = Zettelkasten bidirectional links
- raw/ → wiki/sources/ → wiki/domains/ = PARA's resource processing pipeline
- Evolution layers (lessons → patterns → decisions) = progressive distillation
- Domain indexes = PARA Areas
- Spine/domain overviews = PARA project overview

What's missing:
- Systematic FAQs per domain (quick-reference distillation)
- Comparison matrices (not just prose, but structured tables)
- Regular review cadence (weekly review of new content)
- Personal annotations / reactions layer
- Integration of task management (what to research next → prioritized backlog)

## Key Principles for a Strong Second Brain

1. Capture with context (not just links — capture YOUR takeaways)
2. Atomic units (one idea per note, self-contained)
3. Connection over collection (links matter more than volume)
4. Progressive distillation (raw → summary → insight → action)
5. Review cadence (weekly/bi-weekly to connect new with old)
6. Action orientation (knowledge should lead to decisions/actions)
7. Stable foundation (areas/resources don't churn like projects)
