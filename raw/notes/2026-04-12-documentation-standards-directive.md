## Key Standards for All PM Artifacts

1. **Take time. Do it right. No rushing, cheating, hacking, shortcuts.**
2. **Write for ANYONE — not just yourself.** PM, PO, client, future agent, fresh context.
3. **Every artifact is a handoff document.** If someone picks this up with zero context, can they understand and continue?
4. **Thorough metadata:** custom fields, annotations, header blocks, proper structure.
5. **High standard structure:** indents, title markers, callout types, consistent formatting.
6. **Impediment-ready:** tasks can attach handoff docs for resumption in fresh context.
7. **Preach by example:** the document itself demonstrates the standard it describes.
