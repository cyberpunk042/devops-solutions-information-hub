Build, review, or evolve a wiki model.

Use the model-builder skill (skills/model-builder/skill.md) to:
1. If building new: follow SFIF stages (document → design → scaffold → implement → test)
2. If reviewing: audit against LLM Wiki Standards
3. If evolving: check for new pages/lessons that should be incorporated
4. Always update the super-model (Methodology Framework) when models change
5. Always verify self-referential integrity — the model must pass its own standards
